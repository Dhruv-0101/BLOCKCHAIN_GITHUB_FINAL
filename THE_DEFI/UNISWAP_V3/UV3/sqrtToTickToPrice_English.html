How do you get the tick from sqrtPriceX96?

🧠 STEP 0: REALITY CHECK (VERY IMPORTANT)

In Uniswap V3:

  • ❌ Price is NOT directly stored
  • ❌ Tick is NOT directly updated
  • ✅ Only sqrtPriceX96 changes continuously

Tick is derived from sqrtPriceX96.

🧱 STEP 1: What sqrtPriceX96 Means (REAL FEEL)

Formula:

sqrtPriceX96 = √price × 2^96
Page 2

Example (Real Market):

ETH = $2000

Calculate:

√2000 ≈ 44.72135955
2^96  ≈ 79,228,162,514,264,337,593,543,950,336

So:
sqrtPriceX96 ≈ 44.72135955 × 2^96
             ≈ 3,543,191,142,285,914,378,072,636,784,640  (INTEGER)

👉 This exact value is stored in the pool's slot0

🧠 STEP 2: Tick's Definition (SOURCE OF TRUTH)

Uniswap defines tick as:

price = 1.0001 ^ tick

Reverse this:

tick = log(price) / log(1.0001)

⚠️ But the pool does not know the price directly.

The pool only has sqrtPriceX96.

Page 3

🧮 STEP 3: sqrtPriceX96 → Price (Internally)

The pool internally does this:

price = (sqrtPriceX96 / 2^96)²

Example:

sqrtPriceX96 = 3,543,191,142,285,914,378,072,636,784,640

sqrtPrice = sqrtPriceX96 / 2^96
          ≈ 44.72135955

price = 44.72135955²
      ≈ 2000

👉 The pool has reconstructed the price.

🧮 STEP 4: Price → Tick (Actual Conversion)

Now the tick is calculated:

tick = log(2000) / log(1.0001)

Approximate:

log(2000)   ≈ 7.600902
log(1.0001) ≈ 0.000099995

So:
tick ≈ 7.600902 / 0.000099995
tick ≈ 76009
Page 4

⚠️ On mainnet, the tick has an offset.

That is why you see a tick like ~199320 in the UI.

(because of token decimals + token order)

🧠 STEP 5: How This Works in Real Code

In Uniswap V3 core, this function does the work:

TickMath.getTickAtSqrtRatio(sqrtPriceX96)

Internally it:

  • Takes the sqrtPriceX96
  • Does a log₂ based approximation
  • Then converts it to log₁.₀₀₀₁
  • Returns the nearest valid tick
  • 👉 Pure integer math
  • 👉 Gas efficient
  • 👉 No floating point

  • Every −1 tick = −0.01% price move

  • Price is exponential
